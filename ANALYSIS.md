## 2. Type System (`lib/types/transaction.ts`)

Single source of truth. No `any` anywhere in the codebase.

**Core Types:**
- `RawTransaction` — Post-parse input (invoice_id, date, vendor, amount; optional: category, description, approved_by, address)
- `AnalyzedTransaction` — Full detector output bundle
- `RiskTier` — `'LOW' | 'MEDIUM' | 'HIGH' | 'CRITICAL'`
- `DetectorName` — 12 values: BENFORD_1ST, BENFORD_2ND, ROUND_NUMBER, ISOLATION_FOREST, RSF, EXACT_DUPLICATE, FUZZY_DUPLICATE, SPLIT_INVOICE, DESCRIPTION_AUDIT, EDGAR_UNVERIFIED, OFAC_HIT, ADDRESS_INVALID
- `AnalysisResult` — Full bundle: analyzed transactions + Benford results + portfolio summary
- `PortfolioRiskSummary` — Aggregated metrics (score, tier, rates, flagged count, estimated exposure)

**External Result Types:**
- `EdgarSearchResult` — {cik, entity_name, matched, confidence}
- `OfacCheckResult` — {vendor, hit, matched_name, list_type}
- `NominatimResult` — {address, valid, lat, lon, address_type}

**Type Guards:** `isRawTransaction()`, `isRiskTier()`

## 3. Detector Layer Analysis

### Layer 1 — Statistical

#### `lib/fraud-logic/benford.ts`
- **`analyzeBenfordFirst(transactions)`** — Extracts first non-zero digit from each amount, computes observed vs. Benford expected frequencies, MAD, chi-square. Conformity: MAD < 6% = ACCEPTABLE, 6–10% = MARGINAL, > 10% = NON_CONFORMING.
- **`analyzeBenfordSecond(transactions)`** — Second digit (0–9) for amounts ≥ 10. Same MAD conformity, slightly stricter thresholds (8%, 10%, 12%).
- **`analyzeRoundNumbers(transactions)`** — Flags `amount % 1000 === 0` OR within $15 of approval thresholds [500, 1k, 2.5k, 5k, 9.999k, 10k, 25k, 50k, 100k]. Flagged if round_rate > 15%.
- **`extractBenfordDigits(amount)`** — Per-transaction digit extraction for composite scoring.
- Standards: Nigrini (2012), AICPA AU-C 240, ACFE Fraud Examiners Manual.

### Layer 2 — Pattern

#### `lib/fraud-logic/isolation-forest.ts`
- 200 randomized decision trees (matches scikit-learn v1). Log-transformed + z-score standardized features (amount only). Seeded PRNG (seed=42) for full reproducibility.
- Contamination rate: 0.05 (5% expected outlier rate, per ACFE baseline).
- Output: `risk_scores[]` (0–100), `is_outlier[]`, `raw_scores[]`.
- **Gap:** Only uses `amount` as feature — no vendor, category, or date.
- **Gap:** Contamination hard-coded at 0.05.

#### `lib/fraud-logic/rsf.ts`
- RSF = amount / vendor_median. Flag threshold: RSF > 3.0 (ACFE standard).
- Also computes z-score within vendor distribution.
- Single-transaction vendors: RSF = 1, zscore = 0 (safe default).
- `getTopRsfVendors()` — Top N vendors by total flagged exposure.

#### `lib/fraud-logic/duplicate.ts`
- **Exact:** Composite key `date|amount|vendor` (normalized). Flags groups with count > 1.
- **Fuzzy:** Levenshtein distance ≤ 2 on vendor names/invoice IDs, within ±20% amount band. Lower weight (5%) due to higher false-positive rate.
- `levenshtein(a, b, maxDist=10)` — Classic DP with early exit.
- **Gap:** Fuzzy uses only vendor name + invoice ID, not description text.

#### `lib/fraud-logic/split-invoice.ts`
- Groups by vendor, 7-day rolling window. Flags clusters where individual amounts all < threshold but combined ≥ threshold and ≤ 2× threshold, and amounts within ±15% of max.
- Default thresholds: [1k, 2.5k, 5k, 10k, 25k, 50k, 100k].
- **Gap:** Only catches 7-day structuring — misses slow-bleed over months.
- **Gap:** Thresholds hard-coded, not per-client configurable.

### Layer 3 — Text & External

#### `lib/fraud-logic/description-audit.ts`
- 19 risk patterns (regex), each weighted 15–50 points. Categories: vague/generic, empty/placeholder, high-risk terms (gift, personal, cash advance), correction language.
- Amount weighting: high amounts + vague description → score × 1.3–1.4.
- Missing description → 60 base points + amount multiplier.
- **Gap:** Weights hard-coded; no per-client customization. No NLP.

## 4. Composite Scoring (`lib/fraud-logic/composite-score.ts`)

**Detector Weights (sum to 100):**
| Detector | Weight | Rationale |
|---|---|---|
| ISOLATION_FOREST | 30% | Continuous ML score, lowest false-positive rate |
| RSF | 25% | Vendor-normalized, highly specific |
| EXACT_DUPLICATE | 20% | Binary, near-certain fraud when true |
| SPLIT_INVOICE | 10% | Pattern detection, medium confidence |
| DESCRIPTION_AUDIT | 8% | Text analysis, supplementary |
| FUZZY_DUPLICATE | 5% | Higher false-positive rate |
| ROUND_NUMBER | 2% | Weak individual signal, useful in aggregate |

**Special Cases:**
- OFAC_HIT → automatic score of 100 (immediate CRITICAL)
- RSF 2.0–3.0 (sub-threshold) → partial contribution
- EDGAR_UNVERIFIED + RSF_FLAG → +15 points
- ADDRESS_INVALID → +10 points

**Risk Tiers:** CRITICAL ≥ 80 | HIGH ≥ 60 | MEDIUM ≥ 30 | LOW < 30

**Portfolio Score Formula:**
```
min(outlier_rate × 200, 30) + min(rsf_flag_rate × 150, 25) + min(duplicate_rate × 300, 25) + min(benford_mad × 2, 20)
```

**Gaps:** Weights hard-coded; no calibration UI; no fraud-outcome feedback loop.

## 5. External API Integrations (`lib/external/`)

### `edgar.ts` — SEC EDGAR
- Queries EDGAR Atom feed for company names, computes token-overlap Jaccard similarity. Match threshold: 0.7.
- **Gap:** Token matching crude — misses partial name matches.
- **Gap:** CIK not extracted (returns empty string).
- **Gap:** No client-side rate limiting.

### `nominatim.ts` — OpenStreetMap Geocoding
- Geocodes vendor addresses. Flags: residential, house, apartment, mailbox, virtual_office, place, hamlet.
- Rate-limited to 1 req/sec (OSM policy). Batch is serial with 1.1s sleep between requests.
- **Gap:** 50 vendors = ~50 seconds. No persistent cache.
- **Gap:** No fallback geocoding service.

### `ofac.ts` — US Treasury Sanctions
- Fetches full consolidated SDN XML list. Fuzzy name matching via Levenshtein (distance ≤ 2 or ≤ 5% of max length).
- In-memory cache for Edge Function lifetime only.
- **Gap:** XML parsed via regex — brittle, would break on format changes.
- **Gap:** Soft fail on network error (returns empty list, no alert).

### `app/api/external-verify/route.ts` — Server Proxy
- POST `/api/external-verify` — takes `{vendors[], addresses[]}`, runs EDGAR in parallel, Nominatim serially, returns merged results.
- **Gap:** No retry logic on transient failures. Silently degrades to nulls.

## 6. CSV Parser (`lib/parsers/csv.ts`)

- RFC 4180-compliant. Auto-detects column mapping via aliases for amount, date, vendor, invoice_id, description, address.
- Amount aliases include SAP (DMBTR), Oracle (ENTERED_DR) field names.
- Amount detection fallback: samples 20 rows, picks column with >50% positive numeric values.
- Missing date → defaults to today. Missing vendor → "Unknown". Missing invoice_id → "ROW-{n}".
- **Gap:** Date parsing delegates to JS Date constructor (timezone quirks).
- **Gap:** Only comma-delimited (no semicolon/TSV support).
- **Gap:** Assumes UTF-8 encoding.
- **Note:** PapaParse is listed as a dependency but the custom parser is used instead — PapaParse can be removed.

## 7. Async Pipeline (`lib/fraud-logic/run-async.ts`)

- Wraps `runForensicAnalysis()` with `yieldToUI()` (setTimeout 0) between each detector layer.
- 12 progress steps with `onProgress({step, total, label})` callback.
- After description audit, collects unique vendors + addresses, POSTs to `/api/external-verify`, merges results into `external_results` Map.
- **Gap:** No retry on external API failure — silently falls back to nulls.
